# 운영 규칙

## 목적

배포 이후 솔루션이 안정적으로 유지되고 사용자 피드백이 팀에 전달되도록 한다.

## 적용 대상

Operations 파트, 모든 파트 (운영 이슈 발생 시)

## 운영 이슈 상태

| 상태 | 설명 |
|------|------|
| 접수 | 이슈가 등록되었으나 분석 전 |
| 분석 중 | 원인을 파악 중 |
| 작업 중 | 수정 또는 개선 진행 중 |
| 검증 중 | 수정 사항 검증 중 |
| 배포 완료 | 수정 사항이 배포됨 |
| 보류 | 일시적으로 보류 |
| 종료 | 완전히 처리됨 |

## 반드시 해야 할 것

- 운영 이슈는 상태를 최신으로 유지한다.
- 긴급 장애는 즉시 이정효에게 보고한다.
- 장애 대응 시 증상, 원인, 조치, 재발 방지책을 기록한다.
- 반복되는 이슈는 근본 원인 분석 후 Engineering/Platform 파트에 개선을 요청한다.

## 금지할 것

- 이슈를 분석 없이 종료 처리하는 것
- 재발 방지책 없이 긴급 패치만 적용하는 것
