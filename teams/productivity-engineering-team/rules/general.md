# 일반 규칙

## 목적

모든 에이전트가 공통으로 따라야 할 행동 원칙을 정의한다.

## 적용 대상

Productivity Engineering Team의 모든 파트, 모든 담당자

## 반드시 해야 할 것

- 요청을 받으면 문제를 명확히 이해한 뒤 작업을 시작한다.
- 불명확한 요청은 요청자와 대화하여 목적, 사용자, 제약, 성공 기준을 확인한다.
- 모든 결과물에 요청 배경, 담당자, 작업 내용, 검증 결과, 후속 과제를 기록한다.
- 반복 가능한 업무는 재사용 가능한 형태(문서, 템플릿, 스크립트, skill, plugin)로 만든다.
- 프로젝트 종료 후 운영파트에 인수인계 문서를 작성한다.
- 작업 상태를 변경할 때마다 기록을 남긴다.

## 금지할 것

- 문제 정의 없이 바로 구현을 시작하는 것
- 요청자와 합의 없이 요구사항을 임의로 변경하는 것
- 검증 없이 결과물을 완료로 처리하는 것
- 산출물 위치와 사용 방법을 문서화하지 않는 것

## 예외 상황

- 긴급 장애나 운영 이슈는 즉시 대응을 우선하며, 사후에 문서화한다.
