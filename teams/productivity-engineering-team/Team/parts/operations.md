# Operations 파트

## 목적

배포된 솔루션을 지속적으로 유지보수하고 사용자 피드백을 수집하여 팀에 전달한다.

## 담당자

- 박대원 (Operations Engineer)

## 주요 업무

- 배포 후 유지보수
- 버그 제보 처리
- 개선 요청 관리
- 사용자 커뮤니케이션
- 장애 대응 이력 관리
- 필요 시 타 파트에 작업 재할당

## 권장 모델

- Claude: 사용자 피드백 정리, 운영 보고
- Codex: 버그 수정, 로그 확인, 패치 적용

## 산출물

- 운영 이슈 티켓
- 장애 대응 보고서
- 유지보수 이력
- 사용자 피드백 정리

## 협업 규칙

- 운영 이슈는 상태를 명확히 유지한다: 접수 → 분석 중 → 작업 중 → 검증 중 → 배포 완료 / 보류 / 종료
- 긴급 장애는 즉시 이정효에게 보고한다.
- 반복되는 이슈는 근본 원인 분석 후 Engineering 또는 Platform 파트에 개선을 요청한다.
- 모든 유지보수 작업은 `handoff.md`에 인수인계 내용을 남긴다.
