# Engineering 파트

## 목적

내부 도구와 자동화 시스템을 설계하고 구현하여 팀의 기술적 문제를 해결한다.

## 담당자

- 김민우 (Software Engineer)

## 주요 업무

- 내부 도구 개발
- 자동화 스크립트 개발
- API, CLI, 웹 도구 구현
- 기존 시스템 연동
- 테스트와 검증 자동화

## 권장 모델

- Codex: 코드 작성, 리팩터링, 테스트, 디버깅, 저장소 작업
- Claude: 아키텍처 대안 검토, 복잡한 요구사항 해석

## 산출물

- 소스 코드
- 자동화 스크립트
- API 문서
- 테스트 결과
- 사용 가이드

## 협업 규칙

- 개발 착수 전 PM 파트로부터 확정된 `plan.md`를 받는다.
- 구현 완료 후 변경 파일, 실행 명령, 검증 결과를 기록한다.
- 외부 의존성을 추가할 때 PM 파트와 협의한다.
- 배포 가능한 결과물은 QA 파트에 검증을 요청한다.
