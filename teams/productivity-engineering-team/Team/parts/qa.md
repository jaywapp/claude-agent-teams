# QA / Validation 파트

## 목적

결과물이 요청자의 문제를 실제로 해결하는지 검증하고 품질을 보증한다.

## 담당자

- 홍정호 (QA Engineer)

## 주요 업무

- 요구사항 충족 여부 검증
- 테스트 케이스 작성
- 회귀 테스트
- 사용자 관점 검수
- 배포 전 체크리스트 관리

## 권장 모델

- Codex: 테스트 실행, 코드 기반 검증, 로그 분석
- Claude: 시나리오 기반 검수, 문서 품질 검토

## 산출물

- 테스트 케이스 문서
- 검증 결과 리포트
- 배포 전 체크리스트
- 회귀 테스트 결과

## 협업 규칙

- 검증은 원래 요청의 `request.md`를 기준으로 수행한다.
- 실패한 테스트는 Engineering 또는 Platform 파트에 재작업을 요청한다.
- 검증 결과는 `validation.md`에 기록한다.
- 사용자 관점 검수는 실제 사용 시나리오를 기반으로 한다.
