# team-config: 팀 구성 수정 가이드

이 스킬은 `D:\Agents\productivity-engineering-team` 프로젝트의 에이전트 팀 구성을 수정할 때 따라야 할 절차를 정의한다.

## Codex 플러그인 우선 사용

팀 구성 변경 작업을 시작하기 전에 Codex 플러그인이 설치되어 있는지 확인한다.

**Codex가 있으면:** 파일 수정 작업(멤버 파일 생성, index.md 업데이트, 파트 파일 수정 등)을 직접 수행하지 말고 Codex에게 위임한다. Codex는 저장소 파일 작업에 최적화되어 있기 때문이다.

```
Codex에게 전달할 내용:
- 어떤 파일을 어떻게 수정해야 하는지 (아래 절차 참고)
- 수정 후 변경 내용을 커밋할지 여부
```

**Codex가 없으면:** 아래 절차에 따라 직접 파일을 수정한다.

Codex 플러그인 설치 여부는 현재 세션에서 사용 가능한 도구 목록을 확인하거나 사용자에게 물어본다.

---

## 팀 구조 개요

```
Team/
  purpose.md          # 팀 목표 (직접 수정 자제)
  members/
    index.md          # 담당자 인덱스 및 활성 배정 목록
    {이름}.md         # 개별 담당자 정의
  parts/
    pm.md
    engineering.md
    platform.md
    ax.md
    qa.md
    operations.md
rules/                # 모든 에이전트가 따르는 공통 규칙
```

---

## 작업 유형별 절차

### 1. 활성 담당자 배정 변경 (가장 흔한 작업)

파트에 배정된 담당자를 교체하거나 추가할 때.

**수정할 파일: `Team/members/index.md`**

`## 활성 담당자 배정` 섹션에서 해당 파트의 담당자를 변경한다.

```md
## 활성 담당자 배정

- [[이정효]]: PM / Team Lead
- [[김민준]]: PM / 기획 파트
- [[김민우]]: Engineering 파트        ← 여기서 교체
- [[송주훈]]: Platform / DevOps 파트
- [[정호연]]: AX / AI Automation 파트
- [[홍정호]]: QA / Validation 파트
- [[박대원]]: Operations 파트
```

담당자 풀은 `Team/members/index.md`의 `## 2026 선수단 기준 담당자 풀` 섹션에서 확인한다.
선수단에 없는 이름이 필요하면 아래 "신규 담당자 추가" 절차를 따른다.

---

### 2. 신규 담당자 추가

**체크리스트:**

- [ ] `Team/members/{이름}.md` 파일 생성
- [ ] `Team/members/index.md`의 담당자 풀에 추가
- [ ] 필요하면 활성 배정도 업데이트

**담당자 파일 형식** (`Team/members/{이름}.md`):

```md
# {이름}

- 직책:
- 소속 파트:
- 사용 모델: Claude Sonnet / Claude Opus / Codex
- 주요 역할:
  - 
- 보조 역할:
  - 
- 권한:
  - 
- 산출물:
  - 
- 협업 대상:
  - 
```

**모델 선택 기준:**
- Claude Sonnet: 문서화, 요구사항 정리, 커뮤니케이션, 프롬프트 설계
- Claude Opus: 복잡한 판단, 아키텍처 설계, 복잡한 요구사항 해석
- Codex: 코드 작성, 리팩터링, 테스트, 자동화 스크립트, CI 워크플로

---

### 3. 담당자 역할 수정

**수정할 파일: `Team/members/{이름}.md`**

역할, 권한, 산출물, 협업 대상을 수정한다.
모델 변경이 필요하면 위 기준을 참고한다.

---

### 4. 파트 구조 수정

파트의 업무 범위나 규칙을 변경할 때.

**수정할 파일: `Team/parts/{part-name}.md`**

파트 파일 구조:
- 목적
- 담당자
- 주요 업무
- 권장 모델
- 산출물
- 협업 규칙

파트 담당자 목록을 변경할 경우 `Team/members/index.md`의 활성 배정도 함께 업데이트한다.

---

### 5. 신규 파트 신설

**체크리스트:**

- [ ] `Team/parts/{new-part}.md` 생성 (위 파트 파일 구조 준수)
- [ ] `wiki/{new-part}/index.md` 생성
- [ ] `Team/members/index.md` 활성 배정에 초기 담당자 추가
- [ ] 신규 파트에 배정될 담당자 파일 생성 (필요 시)
- [ ] `start.md`의 섹션 5에 파트 추가 (선택)

---

### 6. 파트 폐지 또는 합병

- 폐지할 파트의 활성 담당자를 다른 파트로 재배정한다.
- `Team/members/index.md` 활성 배정에서 제거한다.
- 파트 파일은 보존하고 상단에 `> **[폐지됨]** YYYY-MM-DD` 표시를 추가한다.
- 해당 파트 wiki 인덱스에도 폐지 표시를 추가한다.

---

### 7. 규칙 수정

`rules/` 하위 파일을 수정한다.

| 파일 | 대상 |
|------|------|
| `general.md` | 전체 공통 원칙 |
| `development.md` | Engineering, Platform, AX 파트 |
| `documentation.md` | 모든 파트의 문서화 기준 |
| `operations.md` | Operations 파트, 운영 이슈 처리 |
| `security.md` | 전체 보안 원칙 |

규칙 파일 형식:
```md
# 규칙 이름
## 목적
## 적용 대상
## 반드시 해야 할 것
## 금지할 것
## 예외 상황
## 관련 문서
```

---

## 주의사항

- `Team/purpose.md`는 팀의 최상위 목표다. 함부로 수정하지 않는다. 수정이 필요하면 요청자와 명확히 합의한 뒤 진행한다.
- 담당자 이름은 `Team/members/index.md`의 선수단 목록에서 가져오는 것이 원칙이다. 선수단에 없는 이름은 이유를 남기고 추가할 수 있다.
- 모든 팀 구성 변경 후 `Team/members/index.md`가 실제 상태를 정확히 반영하는지 확인한다.
