# 운영부 규칙

## 유지보수 범위
- 기존 기능 버그 수정
- 성능 저하 대응
- 보안 패치 적용
- 고객 문의 대응

## 타 부서 업무 재할당 권한
운영부 리드(고종현)는 다음 상황에서 타 부서에 업무를 할당할 수 있다:
- 신규 기능 추가가 필요한 유지보수 → 기획/개발부
- 디자인 수정이 필요한 경우 → 디자인부
- 테스트 재수행이 필요한 경우 → QA부

## 장애 대응 절차
1. 이건희(Support Engineer): 고객 이슈 접수 및 1차 확인
2. 정동윤(SRE): 원인 분석 및 영향도 파악
3. 고종현(Lead): 해결 방향 결정
4. 필요 시 개발부 또는 인프라 담당자 투입
5. 해결 후 포스트모템 작성

## 인시던트 등급
- P1: 서비스 전체 다운 → 즉시 대응 (1시간 내)
- P2: 주요 기능 장애 → 당일 대응
- P3: 부분 기능 이상 → 3일 내 대응
- P4: 경미한 이슈 → 다음 배포 시 반영
