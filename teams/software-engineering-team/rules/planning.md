# 기획부 규칙

## 스펙 문서 형식
`docs/[프로젝트명]/spec.md` 필수 포함 항목:
- 프로젝트 목표 (1~2줄)
- 사용자 페르소나
- 기능 목록 (Must/Should/Could 우선순위)
- 사용자 스토리 (As a [사용자], I want [기능], So that [목적])
- 인수 기준 (Acceptance Criteria)
- 제외 범위 (Out of Scope)

## 요구사항 분석 원칙
- YAGNI: 현재 필요한 기능만 정의한다.
- 모호한 요구사항은 반드시 PM을 통해 고객 확인 후 진행한다.
- 비즈니스 목표와 연결되지 않는 기능은 제안하지 않는다.
