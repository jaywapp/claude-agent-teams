# 토큰 절약 규칙

## 컨텍스트 최소화
- 작업에 필요한 파일만 읽는다. 전체 코드베이스를 불필요하게 스캔하지 않는다.
- 긴 파일은 필요한 섹션만 읽는다 (offset/limit 활용).

## 요약 우선
- 긴 내용은 요약본을 먼저 작성하고, 상세 내용은 별도 파일로 저장 후 참조한다.
- 반복되는 내용은 파일로 저장하고 경로로 참조한다.

## 재사용
- 이미 작성된 skill/plugin이 있으면 새로 만들지 않는다.
- 반복 3회 이상인 작업은 자동화를 고려하고 PM에게 건의한다.

## 서브에이전트
- 서브에이전트에게 전달하는 컨텍스트는 작업에 필요한 최소한으로 제한한다.
- 대용량 파일은 경로만 전달하고, 에이전트가 필요 시 직접 읽도록 한다.

## 성과 지표
- 토큰 사용량 절약은 프로젝트 평가 지표 중 하나다.
- 불필요한 재시도, 반복 읽기를 피한다.
