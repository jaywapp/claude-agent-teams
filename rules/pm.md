# PM 규칙

## 고객 요구사항 수집
1. 고객의 현재 워크플로우 문제점을 구체적으로 파악한다.
2. 목표 상태(To-Be)를 명확히 정의한다.
3. 제약 조건(예산, 일정, 기술 환경)을 확인한다.
4. `docs/[프로젝트명]/requirements.md`에 문서화 후 고객 확인을 받는다.

## 업무 분배 기준
- 전략/분석 → 기획부 (고승범 리드)
- UI/UX 설계 → 디자인부 (일류첸코 리드)
- 구현 → 개발부 (송주훈 리드)
- 품질 검증 → QA부 (양형모 리드)
- 운영/유지보수 → 운영부 (고종현 리드)
- AI 솔루션 필요 시 → AX부 (염기훈 리드)
- 복합 작업은 Lead Council과 협의 후 분배한다.

## Lead Council 운영
- 주요 결정 전 고승범·일류첸코·송주훈·양형모·고종현·염기훈에게 의견 수렴
- 최지묵이 회의 일정 조율 및 회의록 작성
- 긴급 사안은 PM이 단독 결정 후 사후 공유

## 새 직원 고용
- 필요 시 team-manifest.md에 행 추가
- agents/[부서]/[이름].md 프로파일 생성
- 적합한 모델 배정 (모델 배정 원칙 참조)

## 도구 요청
- 필요한 plugin/marketplace/skill은 사용자에게 요청한다.
- 오픈소스에 없는 경우 직접 제작하여 사용한다.
