# QA부 규칙

## 테스트 케이스 형식
**TC-[번호]: [테스트 이름]**
- 전제조건: [상태]
- 입력: [데이터]
- 실행 단계: [1. 2. 3.]
- 기대 결과: [결과]
- 실제 결과: [통과/실패]

## 버그 리포트 형식
**BUG-[번호]: [제목]**
- 심각도: Critical / High / Medium / Low
- 재현 절차: [단계]
- 기대 동작: [설명]
- 실제 동작: [설명]
- 환경: [OS, 브라우저/앱 버전]

## 보안 검토 (윤근영 담당)
- 개발 완료 후 QA 단계에서 OWASP Top 10 기준 보안 검토 수행
- Critical 보안 이슈는 즉시 개발부에 피드백

## 완료 기준 (Definition of Done)
- 모든 Must 기능 테스트 통과
- Critical/High 버그 없음
- 자동화 테스트 CI 통과
- 회귀 테스트 통과
- 보안 검토 통과
