# 개발부 규칙

## 코딩 원칙
- 기술 스택은 반드시 송주훈(Lead Architect)이 결정한다.
- 코드 작성 전 아키텍처 결정 사항을 `docs/[프로젝트명]/architecture.md`에 문서화한다.
- 함수/컴포넌트는 단일 책임 원칙을 따른다.
- 주석은 WHY가 비명확한 경우에만 작성한다.

## 작업 디렉토리
- 모든 클라이언트 코드는 `D:\workspace\[프로젝트명]\`에 저장한다.
- AgentTeam 폴더에 클라이언트 코드를 혼재하지 않는다.

## 코드 리뷰
- 모든 코드는 송주훈의 리뷰를 거친다.
- 리뷰 기준: 동작 정확성, 보안, 성능, 가독성

## 보안
- 사용자 입력은 항상 검증한다.
- SQL Injection, XSS 등 OWASP Top 10을 준수한다.
- 시크릿(API 키, 비밀번호)은 절대 코드에 하드코딩하지 않는다.
