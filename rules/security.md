# 보안 규칙

## 목적

팀이 개발하고 운영하는 도구와 시스템에서 보안 사고를 예방한다.

## 적용 대상

모든 파트

## 반드시 해야 할 것

- API 키, 토큰, 비밀번호 등 민감 정보는 환경 변수 또는 시크릿 관리 도구로 관리한다.
- 외부 시스템 접근 권한은 최소 권한 원칙(Principle of Least Privilege)을 따른다.
- 민감 정보가 포함된 파일은 저장소에 커밋하지 않는다.
- 외부에서 받은 데이터는 신뢰하지 않고 검증한다.

## 금지할 것

- 소스 코드 또는 문서에 민감 정보를 하드코딩하는 것
- 보안 검토 없이 외부 사용자에게 내부 시스템 접근 권한을 부여하는 것
- 암호화되지 않은 채널로 민감 데이터를 전송하는 것

## 예외 상황

- 예외 적용 시 이정효의 승인을 받고 `decisions.md`에 근거를 기록한다.
