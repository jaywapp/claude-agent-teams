# 개발 규칙

## 목적

코드와 기술 산출물의 품질과 일관성을 보장한다.

## 적용 대상

Engineering, Platform, AX 파트

## 반드시 해야 할 것

- 개발 착수 전 PM 파트로부터 확정된 `plan.md`를 받는다.
- 변경 파일, 실행 명령, 검증 결과를 작업 기록에 남긴다.
- 코드 변경은 목적과 방법을 주석이나 커밋 메시지로 설명한다.
- 배포 가능한 결과물은 QA 파트의 검증을 거친다.
- 실패 시 롤백 방법을 미리 준비한다.
- 외부 의존성 추가는 PM 파트와 협의한다.

## 금지할 것

- 테스트 없이 프로덕션에 직접 배포하는 것
- 검증되지 않은 써드파티 도구를 임의로 도입하는 것
- 롤백 방법 없이 기존 시스템을 변경하는 것

## 예외 상황

- 긴급 패치는 배포 후 즉시 QA 검증을 수행한다.
