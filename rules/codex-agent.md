# Codex 에이전트 호출 규칙

## 대상 에이전트 판별

에이전트 프로파일(`.md`) frontmatter의 `model` 필드 값에 `codex`가 포함되면 이 규칙을 따른다. 최신 목록은 [[team-manifest]]에서 확인한다.

## 프롬프트 구성

에이전트의 역할·책임과 규칙 파일 경로만 컨텍스트로 전달하고, 업무를 `## 업무` 섹션으로 이어 붙인다.

```
역할: [직책]
책임: [프로파일의 책임 항목]
규칙: [[rules/general]], [[rules/dev]] (해당 규칙 파일 경로)

## 업무
[작업 내용]
```

## 호출 형식

```
Agent(
  subagent_type="codex:codex-rescue",
  prompt="[위에서 구성한 프롬프트]"
)
```

## 결과물 저장

기존 규칙과 동일하다:
- 임시 업무 문서: `docs/[프로젝트명]/`
- 최종 보고서: `report/`

## 제약

- Codex는 코딩 작업 전용이다. 기획서·디자인·보고서 작성 등 비코딩 업무는 해당 에이전트를 Claude Agent로 호출한다.
- 비코딩 업무가 Codex 에이전트에게 오면 PM이 업무 성격을 판단하여 Claude Agent로 처리한다.
